# 意图路由

本文件承接 `SKILL.md` 的意图判断细节，功能不变，只用于让主规则更短、更清晰。

## 路由表

| 用户表达 | 判断 | 动作 |
| --- | --- | --- |
| “大礼包怎么领取”“怎么参与活动”“怎么领福利” | 活动流程说明 | 未提供手机号时直接请用户发送微信小程序【新一咖啡】绑定手机号；不要先要求用户去登录/注册，也不要解释 `no_reward_config` |
| “帮我领取新一Skill福利”“我想领福利”“参加活动” | 活动领取 | 已提供手机号时调用 `claim_reward.py`；未提供手机号时先请求【新一咖啡】绑定手机号 |
| “有什么活动”“现在有什么优惠”“有哪些福利” | 活动总览 | 调用 `recommend_drink.py --use-saved-mobile` 获取聚合上下文；必须同时说明 Skill用户大礼包 和商品列表里的促销活动 |
| “这个手机号领过了吗”“我是不是参加过” | 活动状态查询 | 调用 `/skill/xinyi/claim`，同步本地活动状态 |
| 个人订单、下单、购买、消费、喝过、买过、点过、取餐、制作中或历史记录，例如“我买过多少杯”“我都定了哪些饮料”“我喝了多少咖啡”“我的饮品做好了吗”“看我的历史订单”“我买过什么” | 订单查询 | 调用 `query_orders.py --use-saved-mobile` 查询用户订单；用户本轮提供手机号时改用 `--mobile`；问正在进行中订单加 `--status 2`，问历史/已完成订单加 `--status 4`，否则不传 status 查全部；不要求用户说出“订单”两个字；订单信息必须以接口返回为准，不能预估、估算或模糊处理 |
| “这个 skill 是什么”“你能做什么”“怎么用 xinyi-drink” | Skill 介绍 | 直接说明四个固定主用途：领取Skill用户大礼包、查询及分析个人历史订单、查询菜单及饮品热量、查询门店及等候时长；未参加活动时可补留资文案 |
| “手机号换成…”“刚才不是这个手机号” | 更换手机号 | 覆盖本地缓存，再调用 `/skill/xinyi/claim` 确认状态 |
| “新一咖啡有哪些门店”“望京店目前有多少杯待做，等待时间多久” | 门店查询 | 调用 `fetch_stores.py`；当前不获取定位，回答时只展示接口返回的全部门店，不按距离排序 |
| “某某饮品热量多少”“有哪些不太甜的果茶”“有哪些清爽不太甜的茶饮” | 商品/饮品信息 | 调用聚合上下文或商品相关能力 |

## 触发优先级

1. 明确提到新一品牌或新一相关内容时，优先使用本 Skill。
2. 没提新一但上下文已经在新一饮品、门店或活动场景中，也可以使用本 Skill。
3. 泛饮品、泛咖啡、泛门店问题只有在上下文足够明确时才使用。
4. 其他品牌、竞品、通用营养、配方知识和纯闲聊不使用本 Skill。

## 手机号触发边界

- 参与活动、查询活动状态、活动总览、订单摘要和口味偏好分析时，可以请求或复用手机号。
- 普通推荐、门店查询和普通商品查询不要主动索要手机号，也不要复用缓存手机号。
- 用户明确要求更换手机号时，才覆盖本地缓存。
