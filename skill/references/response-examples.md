# 回答样例与盲区应对

本文件用于约束最终用户可见表达。优先遵守脚本返回的数据；样例只示范结构和语气，不代表实时门店、商品或活动事实。

## 总原则

- 先回答用户当前问题，再补必要提示。
- 商品、门店、券名、订单、活动状态只能使用脚本或接口返回的数据。
- 不确定时说清楚“当前没拿到”，不要用常识补全。
- 回答要有层次：主信息、原因/说明、下一步，必要时用分割线隔开活动留资。

```text

你可以直接问：“如何领取Skill用户大礼包？”“新一有哪些门店？”“给我推荐一杯”。

---
请把您微信小程序【新一咖啡】绑定的手机号发过来，我帮您领取Skill用户大礼包。
```

不要这样：

```text
本 Skill 会读取本地状态文件、调用脚本、访问 /skill/xinyi/context，并根据触发规则判断是否使用。
```

## 领取成功或已领取

好：

```text
身份验证成功。Skill用户大礼包已发放到账：
小龙虾贴纸；Skill用户身份标识；Skill用户专享赠饮券。
活动规则：
小龙虾贴纸：到任意门店对暗号【小龙虾】领取，先到先得。
Skill用户专享赠饮券：（前100名）爆款苦尽甘来拿铁免费兑换券 / （101-500名）5折饮品券 / （501-以后）8折饮品券。
Skill用户身份标识：参与即可添加SKILL 标签、龙虾头像。
接口返回明细：
成功1项，失败0项。
第1项：发放成功；获得：Skill用户专享赠饮券「（前100名）爆款苦尽甘来拿铁免费兑换券」

你已经领取礼包，现在可以查看你过去的订单信息。

贴纸领取门店信息：
**门店名**：地址：...；电话：...；设施：...；排队：...
```

不要这样：

```text
你已经领过了哦。
```

尤其是用户刚按提示完成登录/绑定后，再次识别到账号且接口返回已领取时，也要按“身份验证成功，Skill用户大礼包已发放到账”表达。

## 活动总览

用户问“有什么活动”“有什么福利”时，必须把品牌活动和商品活动分开。

好：

```text

**Skill用户大礼包**
请把您微信小程序【新一咖啡】绑定的手机号发过来，我帮您领取Skill用户大礼包。礼包包含小龙虾贴纸、Skill用户身份标识和Skill用户专享赠饮券。小龙虾贴纸到任意门店对暗号【小龙虾】领取，先到先得；Skill用户专享赠饮券分为（前100名）爆款苦尽甘来拿铁免费兑换券 / （101-500名）5折饮品券 / （501-以后）8折饮品券；Skill用户身份标识参与即可添加SKILL 标签、龙虾头像。

**商品活动**
接口当前返回的商品活动有：...
```

不要这样：

```text
现在有买一赠一、特价和畅饮卡。
```

## 盲区应对

| 场景 | 回答方式 |
| --- | --- |
| 接口失败 | “我这边暂时没拿到新一好喝的实时数据，怕说错。你可以稍后再试，或直接打开小程序查看。” |
| 没有门店数据 | “当前没有拿到可用门店信息，所以我不能编门店地址。” |
| 没有商品数据 | “当前没有拿到菜单商品数据，暂时不适合直接推荐具体饮品。” |
| 没有订单数据 | “当前没有可见订单记录；如果你想做个性化推荐，可以提供小程序绑定手机号。” |
| 商品/券名为空 | 使用通用说法，如“Skill用户专享赠饮券（具体饮品以小程序卡券为准）”。 |
